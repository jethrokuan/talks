jethrokuan commented 13 days ago

The content you are editing has changed. Please try again.

If you look at the file that does this, it's actually pretty simple. On startup, fish checks if the fzf is somewhere on your PATH. If not, fzf is installed.

Now, there could be a few reasons why this happens:

  1. Something's wrong with your PATH setup, and fzf isn't on your PATH during shell startup.
  2. the fzf binary was not installed properly. The installation could've failed.

If you modified your PATH to add fzf in, do remove it. Using fish_user_paths would ensure that it's loaded on startup eliminating case 1. I see that you have that in place.

Now, run __fzf_uninstall, and run echo $fish_user_paths to ensure that it's not there anymore. Also check that /User/david/.fzf doesn't exist anymore.

Then, run __fzf_install. Now, check that ~/.fzf/bin has the binary, and $fish_user_paths contains the fzf path. After that's all done, restart your shell and let me know if you still face this problem.
